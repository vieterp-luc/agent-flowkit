# Phase 3 — Stage Image Generation (Gemini chained-edit, reverse + chained)

## Context Links
- Parent: `plan.md`, Phase 2 (consumes `analysis.json`)
- Direct pattern: `/Users/vieterp/code/Research/.claude/skills/build-garden/SKILL.md` ("CRITICAL — Stage k chained continuity")
- Memory: `feedback-flow-image-gen-per-chapter` (no mass-batch, sequential to avoid CAPTCHA), `feedback-flow-image-orientation-text` (defensive prompts)

## Overview
Generate N stage images using Google Gemini's chained image-edit:
- **Stage 0** = reverse-engineered from the original input (Gemini removes the finished subject and replaces with the chosen Stage 0 archetype, preserving LAYOUT MAP locked elements like fence/walls/sky for outdoor, or shell for indoor, or surface/bg for objects)
- **Stage 1..N-1** = each generated by feeding **Stage k-1 image** into Gemini with a DELTA prompt (INHERITED / NEW / STILL MISSING bullets)
- **Stage N** = the original input image (no gen)

## Key Insights
- **Gemini, not Flow Imagen.** Reason: Gemini's image-edit endpoint natively takes a prior image + prompt and edits it. Flow Imagen generates from text only (no chained-edit), so consecutive stages would drift wildly. The four `build-*` skills all use Gemini for the same reason.
- Sequential per-stage gen (NOT batch). One stage at a time prevents reCAPTCHA + lets us probe quota after each.
- LOCK BLOCKS pasted verbatim inside every stage prompt — the user sees only the user-facing summary, not the LOCK blocks.

## Requirements

### Functional
- Generate stage_00.jpg first (Stage 0 reverse).
- Then sequentially generate stage_01..stage_(N-1).jpg, each chained from previous output.
- Copy original input → stage_<N>.jpg.
- Save each stage prompt to `prompts/stage_<NN>_image_prompt.txt`.
- Verify each generated image: file exists, > 30KB, resolution is 9:16 (or square then crop center-to-9:16).
- On QUOTA / failure: halt, save state, print resume instructions.
- `--resume` skips stages where `stages/stage_<NN>.jpg` already exists.

### Non-functional
- Time budget: ≤ 90s per stage gen → ≤ 6 min for N=4.
- Gemini Lyria / Imagen-edit uses the same `gemini` CLI bootstrap (Playwright) already wired for Lyria (see `fk-gen-music.md` Path A).

## Architecture

### Gemini invocation path
Two options — pick **Option A** (CLI) for KISS:

#### Option A — `gemini` CLI (recommended)
```bash
echo "<prompt>" | gemini -y -m gemini-2.5-flash-image \
  --image input.jpg --image-output stages/stage_NN.jpg
```
- Already used elsewhere in the repo (`ai-multimodal` skill); user already bootstrapped login.
- Stateless per call — perfect for sequential chained gen.
- Output forced to 9:16 by appending `vertical 9:16 portrait 1080x1920` to prompt.

#### Option B — server endpoint
- Would require adding new `/api/gemini/browser/edit-image` endpoint mirroring the music one. YAGNI for v1; keep as future opt-in.

### Stage 0 prompt template
```
[VERTICAL PORTRAIT 9:16, 1080x1920]
Transform this image so the {{subject_display}} is COMPLETELY removed and
replaced with {{stage_0_archetype_description}}.

LOCKED — keep IDENTICAL to original:
- Camera angle, focal length, framing
- Sky / weather / time-of-day / ambient lighting
- {{locked_surroundings_per_subject}}
  · house/building: fence/wall boundary, neighboring buildings, external trees,
    sidewalk/street pattern
  · garden: fence/wall, neighboring overhanging trees, building edges visible at frame border
  · room: room SHELL — walls, ceiling, floor SUBFLOOR, window/door openings, architectural features
  · object: SURFACE the object rests on, background, table/easel/board, lighting setup

{{LAYOUT_MAP}}  ← paste verbatim from analysis.json

Stage 0 must look RADICALLY DIFFERENT from input — wasteland/raw-materials
character dominates 80-90% visual weight; future layout hints ≤ 10%.
{{SUBJECT_SPECIFIC_STAGE_0_RULES}}

STRICTLY NO TEXT, NO TYPOGRAPHY, NO WATERMARKS, NO LOGOS.
Photorealistic, ultra-detailed, cinematic. IDENTICAL camera angle to input.
```

`SUBJECT_SPECIFIC_STAGE_0_RULES` examples:
- house: "Show the property in DECAYED state — peeling paint, broken windows, overgrown yard. Lot dimensions and house footprint preserved."
- garden: "Apply build-garden's Stage 0 rule — pond pit visible if final has pond, otherwise wasteland surface dominates."
- room: "Show bare empty shell — stripped walls, bare subfloor, single bulb hanging, NO furniture."
- building: "Show empty bare plot of graded earth with temp construction fencing, materials staged on side, no foundation poured yet."
- object: "Show ONLY the raw materials + tools needed to make this object, arranged on the same surface as input. The finished object MUST be absent. The surface, background, and lighting stay IDENTICAL."

### Stage k prompt template (k = 1..N-1)
```
[VERTICAL PORTRAIT 9:16, 1080x1920]
Starting from this image (Stage {{k-1}} of {{subject_display}} build,
showing {{brief_recap_stage_k_minus_1}}), advance it to Stage {{k}} —
approximately {{percent}}% complete at the "{{milestone_name}}" milestone.

LOCKED — keep IDENTICAL to Stage {{k-1}}:
- Camera angle, focal length, framing
- Sky / lighting / surroundings
- {{locked_surroundings_per_subject}}
- Everything already built in Stage {{k-1}}

INHERITED from Stage {{k-1}} (must look IDENTICAL at these elements):
- {{list_inherited}}

NEW IN Stage {{k}} (added/changed in this transformation):
- {{list_new}}

STILL MISSING in Stage {{k}} (will appear in later stages — DO NOT add yet):
- {{list_missing}}

{{LAYOUT_MAP}}    ← verbatim

{{INVENTORY_LOCK}}  ← verbatim
  (subject-specific name: PLANT INVENTORY / FURNITURE INVENTORY /
   MATERIAL INVENTORY / OBJECT/COMPONENT INVENTORY)

CONTINUITY RULES:
- Position/orientation LOCK: every zone at EXACT same position as LAYOUT MAP.
- Identity LOCK: every plant/component/material at the SAME name + spec as
  INVENTORY — only completion/maturity/finish changes per stage.
- Camera LOCK: never move.

STRICTLY NO TEXT, NO TYPOGRAPHY, NO WATERMARKS, NO LOGOS.
Photorealistic, ultra-detailed.
```

Stored inline in the skill (no external `stage-milestones.md` for v1 — KISS):

Adjust proportionally for N=2..8 (compress / expand).

### Quota probe pattern
```
1. Generate Stage 0 → if FAIL with QUOTA-like error → halt + emit resume hint
2. If Stage 0 OK → generate Stage 1; halt on quota
3. ... etc.
```

## Success Criteria
- For all 5 subjects: Stage 0 reads as "dramatic before" (passes the 20% → 100% rule).
- Stage k → Stage k+1 visibly progresses one milestone without species/component/material drift.
- All N stages have IDENTICAL camera angle (eyeball check).
- All stages are 9:16 1080×1920 or ratio-correct after auto-crop.

## Risk Assessment
- Risk: Gemini drops camera lock mid-chain. Mitigation: re-paste LAYOUT MAP + INVENTORY in every prompt; if drift detected, regenerate with stronger wording.
- Risk: Object subtypes (esp. food, painting) hallucinate components not in INVENTORY. Mitigation: explicit "DO NOT introduce components not listed in INVENTORY" line at end of every prompt.
- Risk: 9:16 cropping cuts important content. Mitigation: instruct user (in skill output) to feed input image that is already 9:16 or has content centered; auto-crop center-9:16 with warning.

## Security Considerations
- Stage images may inadvertently include readable text from input — guardrail "STRICTLY NO TEXT" repeated. If input had a sign/logo, Stage 0 must replace it with blank.
